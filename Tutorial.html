Tuner Tips

DC spike at center is hardware leakage - not a real signal

Gain too high -> noise floor rises, weak signals get buried

2.4 MSPS = widest view but highest aliasing risk

A strong nearby signal can desensitize the whole tuner

Blackman-Harris window reduces spectral leakage between bins

Twiddle Factor Guide

SliderWhat it doesTune it when...
Avg alpha Controls how quickly the spectrum curve reacts. Blends each new FFT frame into a running average — low alpha holds onto old frames longer. Raise it to see fast-moving signals clearly. Lower it to smooth out a noisy baseline.
Smooth sigma Applies a gaussian blur across FFT bins before peak detection. Wider sigma merges neighboring bins together. Raise it to kill narrow noise spikes. Lower it (or zero) if you need to resolve two close signals.
Prominence How far a peak must rise above its immediate surroundings to count. Prevents noise bumps from being flagged as signals. Raise it if you are seeing too many false peaks. Lower it if real but weak signals are being missed.
Min height Absolute dBm floor. Any peak below this level is ignored outright, regardless of how prominent it looks locally. Raise it (less negative) to ignore deep-noise detections. Lower it if you want to chase weak signals.
Peak BW Minimum width a peak must span in kHz to be counted. Narrow noise spikes are excluded; broad signals like FM pass through. Lower it to detect narrow signals like pagers or POCSAG. Keep it near 200 kHz for FM-only work.
Sweep dwell How many FFT frames to average at each sweep step before moving on. One frame is ~1.7 ms of data. More frames = cleaner average, slower pass. 3 frames is fine for FM and other continuous signals. Raise to 10-20 for bursty signals like ADS-B or ISM devices that only transmit for microseconds.
